How it works

SVG path morphing requires that both the start and end d attribute have the same number and types of path commands. When command counts match, the browser (and GSAP) can linearly interpolate each numeric parameter independently.

All four shapes here — circle, star, blob, hexagon — are built with exactly 6 cubic bezier (C) commands plus a closing Z. GSAP's attr tween interpolates the raw string values, and the gradient stops animate simultaneously for a synchronized color shift.

1

Design matching paths — Every morph target must have identical command sequences. "M → C C C C C C Z" must be the same structure in all shapes, only coordinates differ.

2

GSAP attr tween — gsap.to(path, { attr: { d: newPath } }) tells GSAP to treat the path string as a set of interpolatable numbers.

3

Chain with delayedCall — onComplete: () => gsap.delayedCall(0.8, morphNext) creates a staggered loop with pause between transitions.

4

Animate gradient stops simultaneously — Color shifts sync with shape changes by tweening the stop-color attribute on gradient stops in parallel.
