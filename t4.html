How it works

Both charts are triggered by ScrollTrigger: they animate in when the user scrolls them into view and reverse out when scrolled past. This makes re-animation automatic on re-scroll, perfect for presentations and demos.

The bar chart uses CSS transform: scaleY() starting from 0. This is GPU-composited — no repaints, no layout shifts. Counter values use a GSAP object tween pattern: animate a plain JS object's property and read it in onUpdate.

The line chart uses the same stroke-dashoffset trick as the circuit demo: measure the path length, set it as both dasharray and offset, then tween offset to zero. Data points pop in with back.out easing after the line finishes.

1

ScrollTrigger timeline — Wrapping both bar and counter animations in a single timeline ensures they start together and can reverse cleanly.

2

GPU-composited scaleY — CSS transforms run on the compositor thread. Zero bars start with transform: scaleY(0) set at authoring time, not injected by JS.

3

Object counter pattern — gsap.to(counterObj, { val: 42, onUpdate: () => el.textContent = Math.round(obj.val) + 'M' }) animates any number with any ease.

4

Line chart path drawing — path.getTotalLength() + dashoffset tween with a staggered dot reveal creates a satisfying "plotting" effect.
