☀ Orbital System

Motion Along Path

Planets orbit a central sun using GSAP's MotionPathPlugin. Each world follows an invisible SVG ellipse at its own speed and inclination.

MotionPathPlugin SVG ellipse paths staggered timelines

How it works

Each planet is a bare <circle> element with no initial position. GSAP's MotionPathPlugin takes an SVG element as the path and animates another element's position along it. The ellipses are real SVG elements in the document but rendered with near-zero opacity so they're invisible yet fully usable as paths.

Trail circles lag behind each planet by 6% and 12% of the orbit duration using start: offset - lag. Combined with reduced opacity, this creates the illusion of a glowing comet trail without any canvas or custom drawing code.

1

Reference SVG ellipses as paths — motionPath: { path: '#orbit1', align: '#orbit1' } tells GSAP to animate along the ellipse and keep the element centered on it.

2

start/end offset for staggered entry — Each planet begins at a different progress point (0, 0.25, 0.5, 0.75) so the system looks populated from the start, not all launching from the same spot.

3

Trail via lagged copies — Two additional circles per planet follow the same path but with start - 0.06 and start - 0.12 offset, plus reduced opacity for depth.

4

Sun pulse — The central sun uses a simple scale: 1.15 yoyo tween for the breathing glow effect, composited by the feGaussianBlur filter.
