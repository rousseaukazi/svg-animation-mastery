feTurbulence + feDisplacementMap warps geometry in real-time

RGB Glitch

feOffset separates R/G/B channels randomly, snapping back to baseline

How it works

SVG filters are a pipeline of primitives applied to a source graphic. Each <fe*> element takes an input, processes it, and passes the result to the next stage. The entire pipeline runs on the GPU in most browsers.

The liquid text effect uses feTurbulence to generate organic noise, then feDisplacementMap to warp the text pixels using the noise's R and G channels as X and Y displacement respectively. Animating baseFrequency via rAF makes it flow continuously.

The RGB glitch splits the source into three color channels using feColorMatrix, randomly offsets each with feOffset, then re-merges them with feMerge. The burst is triggered by a GSAP delayedCall loop with randomized intervals.

1

Liquid text pipeline — feTurbulence → feDisplacementMap. The turbulence baseFrequency is updated every rAF frame using sinusoidal easing for smooth flow.

2

Glow pulse pipeline — feGaussianBlur → feColorMatrix (tint to green) → feMerge (blur under source). GSAP tweens stdDeviation between 6 and 20.

3

RGB glitch pipeline — Three parallel feOffset → feColorMatrix branches, one per color channel (R, G, B), merged with feMerge. Each gets a random dx/dy on glitch frames.

4

Glitch timing — A recursive gsap.delayedCall(0.8 + Math.random() * 2.5, triggerGlitch) schedules bursts at unpredictable intervals for realism.
